# 审批管理修复 - 交付总结

## 问题描述

**用户反馈**：
> 审批管理页面这里，有些高级权限的人处理了全部待审的，然后处理完的那条不在我已处理那里

**问题分析**：
- 高级权限用户（如 CFO、财务经理）处理完待审单据后
- 这些单据没有出现在"我已处理"标签页中
- 导致用户无法查看自己的审批历史记录

## 根本原因

在 `routes/approvals.py` 文件中，审批动作处理逻辑存在缺陷：

```python
# 原有代码（有问题）
if not _safe_text(row.get("first_approver_id")):
    updates["first_approver_id"] = actor
if not _safe_text(row.get("first_approved_at")):
    updates["first_approved_at"] = now_text
```

**问题**：
- 只有当 `first_approver_id` 字段为空时才记录审批人
- 如果字段已有值（即使是其他人或旧数据），就不会更新
- 导致实际处理人没有被正确记录

## 修复方案

### 修改内容

**文件**：`routes/approvals.py`  
**位置**：第 389-410 行

**修复后的代码**：
```python
# 低/中风险单据：一次性通过
else:
    updates["approval_stage"] = "DONE"
    updates["approval_status"] = "APPROVED"
    updates["status"] = "APPROVED"
    updates["queue_owner_id"] = ""
    # 始终记录当前审批人
    updates["first_approver_id"] = actor
    updates["first_approved_at"] = now_text

# 退回/驳回
elif action in {"REJECT", "RETURN"}:
    if not comment:
        return jsonify({"ok": False, "msg": "comment is required for reject/return"}), 400
    target_status = "REJECTED" if action == "REJECT" else "RETURNED"
    updates["approval_stage"] = "DONE"
    updates["approval_status"] = target_status
    updates["status"] = target_status
    updates["queue_owner_id"] = ""
    # 始终记录当前审批人
    updates["first_approver_id"] = actor
    updates["first_approved_at"] = now_text
```

### 关键改变

1. **移除条件判断**：不再检查 `first_approver_id` 是否为空
2. **始终记录**：每次审批动作都记录当前操作人
3. **保持一致**：通过、退回、驳回三种动作使用相同的记录逻辑
