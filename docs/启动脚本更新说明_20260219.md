# 启动脚本更新说明 - 2026-02-19

## ✅ 已修复的问题

### 1. 保留管理员账号 ✅
**问题**: 启动脚本删除数据库后，管理员账号（admin01）也被删除，导致无法登录

**解决方案**:
- 脚本会自动检测 `database.db` 中是否存在 `admin01` 账号
- 如果存在，会自动复制到新创建的 `database_demo.db` 或 `database_clean.db`
- 保留完整的账号信息（用户名、密码、角色、权限等）

**实现逻辑**:
```batch
REM 检查是否存在 admin01 用户
if exist "database.db" (
    检测 admin01 账号
    如果存在 -> 复制到新数据库
)
```

### 2. 自动打开浏览器 ✅
**问题**: 启动后需要手动打开浏览器访问 http://127.0.0.1:5000

**解决方案**:
- Flask 启动后，3秒自动打开默认浏览器
- 无需手动复制粘贴网址

**实现代码**:
```batch
REM 延迟3秒后自动打开浏览器
start "" cmd /c "timeout /t 3 /nobreak >nul && start http://127.0.0.1:5000"
```

### 3. 跳过手动确认 ✅
**问题**: 每次启动都需要手动输入 `Y` 确认

**解决方案**:
- 显示提示信息后，自动倒计时 3 秒
- 无需手动输入，自动继续

**实现代码**:
```batch
echo [自动确认] 3秒后自动继续...
timeout /t 3 /nobreak >nul
echo OK 已确认，继续启动
```

## 🔐 管理员账号保护机制

### 工作原理

1. **检测阶段**
   - 脚本启动时检查 `database.db` 是否存在
   - 查询是否有 `username='admin01'` 的用户

2. **复制阶段**
   - 如果检测到 admin01，从 `database.db` 读取完整用户信息
   - 复制到新创建的 `database_demo.db` 或 `database_clean.db`
   - 保留所有字段：密码哈希、角色、权限、部门等

3. **验证阶段**
   - 复制成功后显示确认信息
   - 可以使用原来的账号密码登录

### 支持的字段

复制的用户信息包括：
- `id` - 用户ID
- `username` - 用户名（admin01）
- `password_hash` - 密码哈希
- `department` - 部门
- `employee_name` - 员工姓名
- `employee_no` - 员工编号
- `role` - 角色
- `status` - 状态
- `must_change_password` - 是否需要修改密码
- `failed_login_attempts` - 失败登录次数
- `lock_until` - 锁定截止时间
- `password_updated_at` - 密码更新时间
- `position_id` - 职位ID

## 🔧 技术细节

### 管理员账号复制 SQL

```python
# 从 database.db 读取 admin01
src = sqlite3.connect('database.db')
src_cursor = src.execute('SELECT * FROM users WHERE username="admin01"')
row = src_cursor.fetchone()

# 写入到 database_demo.db
dst = sqlite3.connect('database_demo.db')
dst.execute('DELETE FROM users WHERE username="admin01"')
dst.execute('INSERT INTO users VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)', row)
dst.commit()
```

## ⚠️ 注意事项

1. **首次使用**
   - 如果 `database.db` 不存在或没有 admin01 账号
   - 新数据库将不会有管理员账号
   - 需要手动创建或使用默认账号

2. **密码安全**
   - 复制的是密码哈希，不是明文密码
   - 原密码保持不变

3. **数据隔离**
   - `database.db` 永远不会被删除
   - 只操作 `database_demo.db` 和 `database_clean.db`

4. **浏览器打开**
   - 使用系统默认浏览器
   - 如果 Flask 启动失败，浏览器会显示无法连接

## 📞 故障排除

### 问题1: 管理员账号没有被复制

**可能原因**:
- `database.db` 不存在
- `database.db` 中没有 admin01 用户
- 用户表结构不匹配

**解决方法**:
1. 检查 `database.db` 是否存在
2. 使用 SQLite 工具查看是否有 admin01 用户
3. 手动创建管理员账号

**可能原因**:
- Flask 启动失败
- 系统没有默认浏览器

**解决方法**:
1. 查看终端错误信息
2. 手动访问 http://127.0.0.1:5000
3. 检查端口 5000 是否被占用

**解决方法**:
修改脚本中的倒计时时间：
