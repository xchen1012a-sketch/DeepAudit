# DeepAudit Pro 启动脚本使用说明

## 📍 脚本文件位置

所有启动脚本都位于项目的 `scripts` 目录下：

```
C:\Users\画桦\Desktop\DeepAudit_Pro\scripts\
├── run_demo.bat      # 演示模式启动脚本
├── run_clean.bat     # 纯净模式启动脚本
└── seed_demo.py      # 演示数据生成脚本
```

## 🚀 使用方法

### 方法一：双击运行（推荐）

1. 打开文件资源管理器
2. 进入 `C:\Users\画桦\Desktop\DeepAudit_Pro\scripts` 目录
3. 双击 `run_demo.bat` 或 `run_clean.bat`

### 方法二：命令行运行

打开 CMD，执行以下命令：

```cmd
# 演示模式
C:\Users\画桦\Desktop\DeepAudit_Pro\scripts\run_demo.bat

# 纯净模式
C:\Users\画桦\Desktop\DeepAudit_Pro\scripts\run_clean.bat
```

### 方法三：从项目根目录运行

```cmd
cd C:\Users\画桦\Desktop\DeepAudit_Pro
scripts\run_demo.bat
```

## 📋 启动流程

### 演示模式 (run_demo.bat)

```
[1/8] 检查/创建虚拟环境 (.venv)
[2/8] 激活虚拟环境
[3/8] 安装依赖 (requirements.lock.txt 或 requirements.txt)
[4/8] 设置环境变量 (DATABASE_URL=sqlite:///database_demo.db)
[5/8] 用户确认（输入 Y 继续）
[6/8] 删除 database_demo.db（如存在）
[7/8] 初始化数据库结构
[8/8] 生成演示数据 (scripts\seed_demo.py)
[9/9] 启动 Flask (http://127.0.0.1:5000)
```

### 纯净模式 (run_clean.bat)

```
[1/8] 检查/创建虚拟环境 (.venv)
[2/8] 激活虚拟环境
[3/8] 安装依赖 (requirements.lock.txt 或 requirements.txt)
[4/8] 设置环境变量 (DATABASE_URL=sqlite:///database_clean.db)
[5/8] 用户确认（输入 Y 继续）
[6/8] 删除 database_clean.db（如存在）
[7/8] 初始化数据库结构（空白）
[8/8] 跳过演示数据生成
[9/9] 启动 Flask (http://127.0.0.1:5000)
```

## 🔒 安全保证

### 数据库隔离

| 数据库文件 | 用途 | 是否会被删除 |
|-----------|------|-------------|
| `database.db` | 生产数据 | ❌ **绝不删除** |
| `database_demo.db` | 演示数据 | ✅ 每次启动删除 |
| `database_clean.db` | 开发数据 | ✅ 每次启动删除 |

### 确认提示

每次启动前都会显示：
```
========================================
  WARNING 重要提示
========================================

演示模式将执行以下操作：
  1. 删除并重建 database_demo.db
  2. 自动生成演示数据（审计日志、风险事件、发票台账）
  3. 不会影响 database.db（如存在）

目标数据库：database_demo.db
数据隔离：完全独立，不影响生产数据

确认启动演示模式？(输入 Y 继续，其他键取消):
```

## 🐛 常见问题

### Q1: 脚本显示乱码怎么办？
**A**: 已修复！新版本使用简单字符（X、OK、WARNING），不会出现乱码。

### Q2: 提示找不到 requirements.txt？
**A**: 确保在项目根目录有 `requirements.txt` 或 `requirements.lock.txt` 文件。

### Q3: 虚拟环境激活失败？
**A**: 检查 Python 是否正确安装，运行 `python --version` 确认。

### Q4: 数据库初始化失败？
**A**: 检查是否有其他程序占用数据库文件，关闭后重试。

### Q5: Flask 启动失败？
**A**: 查看错误信息，可能是端口 5000 被占用，或依赖未正确安装。
