# 🐛 启动脚本调试指南

## 问题：脚本闪退

如果启动脚本（`run_demo.bat` 或 `run_clean.bat`）闪一下就消失了，说明脚本执行过程中出错了。

---

## 🔍 使用调试版本查找错误

我已经创建了两个调试版本的脚本，它们会：
1. 记录详细的执行日志
2. 出错时不会闪退，会暂停显示错误
3. 生成日志文件供查看

### 调试版本脚本

| 原脚本 | 调试版本 | 日志文件 |
|--------|---------|---------|
| `run_demo.bat` | `run_demo_debug.bat` | `run_demo_debug.log` |
| `run_clean.bat` | `run_clean_debug.bat` | `run_clean_debug.log` |

## 🔧 常见错误及解决方法

### 错误 1: Python 未安装

**错误信息**：
```
X Python 未安装或不在 PATH 中
```

**解决方法**：
1. 安装 Python 3.8 或更高版本
2. 安装时勾选 "Add Python to PATH"
3. 或手动添加 Python 到系统 PATH

**验证**：
```cmd
python --version
```

---

### 错误 2: 虚拟环境创建失败

**错误信息**：
```
X 虚拟环境创建失败
```

**解决方法**：
1. 确保有足够的磁盘空间
2. 确保项目目录没有中文路径问题
3. 手动创建虚拟环境：
```cmd
cd C:\Users\画桦\Desktop\DeepAudit_Pro
python -m venv .venv
```

---

### 错误 3: 依赖安装失败

**错误信息**：
```
X 依赖安装失败
```

**解决方法**：
1. 检查网络连接
2. 手动安装依赖：
```cmd
cd C:\Users\画桦\Desktop\DeepAudit_Pro
.venv\Scripts\activate
pip install -r requirements.lock.txt
```

3. 如果还是失败，尝试：
```cmd
pip install --upgrade pip
pip install -r requirements.lock.txt
```

---

### 错误 4: 数据库初始化失败

**错误信息**：
```
X 数据库初始化失败
```

**可能原因**：
- 数据库文件被占用
- 导入模块失败
- 代码错误

**解决方法**：
1. 关闭所有可能占用数据库的程序
2. 查看日志文件中的详细错误信息
3. 手动测试初始化：
```cmd
cd C:\Users\画桦\Desktop\DeepAudit_Pro
.venv\Scripts\activate
python -c "from core.app_factory import create_app; from utils.db import init_db; app = create_app(); app.app_context().push(); init_db()"
```

---

### 错误 5: Flask 启动失败

**错误信息**：
```
X 服务器启动失败
```

**可能原因**：
- 端口 5000 被占用
- 代码错误
- 配置错误

**解决方法**：
1. 检查端口占用：
```cmd
netstat -ano | findstr :5000
```

2. 如果端口被占用，结束占用进程或使用其他端口：
```cmd
set FLASK_RUN_PORT=5001
flask run
```

3. 查看日志文件中的详细错误

   ```cmd

   ```

4. **根据错误信息查找解决方法**
   - 参考上面的常见错误
   - 或将错误信息发给我

5. **修复问题后重新测试**
