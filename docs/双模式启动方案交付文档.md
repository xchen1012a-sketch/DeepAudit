- 三个数据库文件互不影响
- 通过环境变量 `DATABASE_URL` 切换

## 三、技术实现细节

### 3.1 应用层数据库切换支持

**修改位置**: `utils/db.py` (第 21-42 行)

**实现逻辑**:
```python
def _get_db_path_from_env() -> str:
    """从环境变量解析数据库路径，优先级：DATABASE_URL > DB_PATH > 默认值"""
    # 1. 优先读取 DATABASE_URL（SQLAlchemy 格式：sqlite:///xxx.db）
    database_url = os.getenv("DATABASE_URL", "").strip()
    if database_url and database_url.startswith("sqlite:///"):
        db_file = database_url[10:]  # 去掉 "sqlite:///"
        if db_file:
            # 相对路径相对于项目根目录
            if not os.path.isabs(db_file):
                return os.path.abspath(str(PROJECT_ROOT / db_file))
            return os.path.abspath(db_file)
    
    # 2. 兼容旧的 DB_PATH 环境变量
    db_path = os.getenv("DB_PATH", "").strip()
    if db_path:
        return os.path.abspath(db_path)
    
    # 3. 默认值
    return os.path.abspath(str(PROJECT_ROOT / "database.db"))

DB_PATH = _get_db_path_from_env()
```

**特点**:
- ✅ 支持 SQLAlchemy 标准格式 `sqlite:///database.db`
- ✅ 支持相对路径和绝对路径
- ✅ 向后兼容旧的 `DB_PATH` 环境变量
- ✅ 无硬编码，完全动态

### 3.2 演示模式启动脚本

**文件**: `scripts/run_demo.bat`

**执行流程**:
```
1. 检查/创建虚拟环境 (.venv)
2. 激活虚拟环境
3. 安装依赖 (requirements.lock.txt 或 requirements.txt)
4. 设置环境变量:
   - DATABASE_URL=sqlite:///database_demo.db
   - APP_MODE=demo
   - FLASK_DEBUG=1
   - DEV_ALLOW_INSECURE=1
5. 显示确认提示（中文，醒目）
6. 删除 database_demo.db（如存在）
7. 初始化数据库结构 (flask db upgrade 或 init_db)
8. 执行 scripts/seed_demo.py 生成演示数据
9. 启动 Flask (127.0.0.1:5000)
```

**安全特性**:
- ✅ 启动前显示醒目提示，说明将删除 `database_demo.db`
- ✅ 要求用户输入 `Y` 确认
- ✅ 只删除 `database_demo.db`，绝不删除 `database.db`
- ✅ 全程中文输出，清晰易懂

## 七、常见问题

### Q1: 如何切换到生产数据库？

**A**: 手动设置环境变量：
```cmd
set DATABASE_URL=sqlite:///database.db
set APP_MODE=production
flask run
```

### Q2: 演示数据可以修改吗？

**A**: 可以修改 `scripts/seed_demo.py`，添加或修改演示数据。

### Q3: 如何备份数据库？

**A**: 直接复制数据库文件：
```cmd
copy database.db database.db.backup
```

### Q4: 启动脚本报错怎么办？

**A**: 检查以下几点：
1. Python 是否正确安装
2. 虚拟环境是否正常
3. 依赖是否安装完整
4. 数据库文件是否被占用

### Q5: 如何在 Linux/Mac 上使用？

**A**: 需要将 `.bat` 脚本改写为 `.sh` 脚本，逻辑相同。
