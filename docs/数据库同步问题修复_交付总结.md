## 技术说明

### 为什么会出现多个数据库文件？

1. **`database.db`（根目录）**：
   - 配置默认值 `sqlite:///database.db` 解析为根目录
   - Flask 应用实际写入此文件

2. **`instance/database.db`**：
   - Flask 的默认实例目录
   - 如果配置不当可能会创建此文件
   - 本项目中可能未使用

3. **`exports/database_real_for_navicat.db`**：
   - 由 `export_real_db.py` 创建的静态副本
   - 用于在不影响应用运行的情况下查看数据
   - 不会自动同步，需手动导出

### SQLite 文件锁机制

SQLite 使用文件锁保证数据一致性：
- **读锁**：多个进程可以同时读
- **写锁**：只有一个进程可以写，且会阻塞其他读写

因此：
- Flask 运行时（有写操作），Navicat 可能无法打开同一文件
- Navicat 打开时（持有读锁），Flask 写入可能被阻塞

**解决方案**：
- 方案 A：交替使用（关闭一个再打开另一个）
- 方案 B：使用导出副本（可以同时运行）

### 相对路径解析规则

`sqlite:///database.db` 的解析：
- 相对于**项目根目录**（`PROJECT_ROOT`）
- 不是相对于当前工作目录（`cwd`）
- 由 `utils/db.py` 的 `_get_db_path_from_env()` 处理

示例：
```python
# 配置
SQLALCHEMY_DATABASE_URI = "sqlite:///database.db"

# 解析为
C:\Users\画桦\Desktop\DeepAudit_Pro\database.db
```

## 常见问题

### Q1: 为什么 Navicat 看不到最新数据？

**A**: 检查以下几点：
1. Navicat 打开的文件路径是否正确（运行 `diagnose_db_path.py` 确认）
2. 如果使用导出副本，是否重新运行了导出脚本
3. Navicat 是否刷新了连接

### Q2: 如何确认 Flask 写入的是哪个数据库？

**A**: 运行 `python scripts\verify_db_write.py`，会显示写入目标并执行测试写入。

### Q3: 可以同时运行 Flask 和 Navicat 吗？

**A**: 
- **方案 A（直接打开真实库）**：不可以，SQLite 文件锁冲突
- **方案 B（使用导出副本）**：可以，但需要手动同步

### Q4: 如何切换数据库位置？

**A**: 在项目根目录创建 `.env` 文件，设置 `DATABASE_URL`：
```env
DATABASE_URL=sqlite:///D:/MyData/deepaudit.db
```

### Q5: 如何删除多余的数据库文件？

**A**: 
1. 先运行 `diagnose_db_path.py` 确认哪个是真实库
2. 备份真实库
3. 删除其他副本（如 `instance/database.db`）
4. 保留 `exports/` 下的副本作为导出目标
