# 发票验真服务对接指南

## 一、真实发票验真规则

### 1. 验真通过条件

发票必须同时满足以下所有条件：

✅ **发票存在且有效**
- 发票代码和号码在税务系统中存在
- 发票状态为"正常"（NORMAL）
- 未被作废（VOID）
- 未被红冲（RED）

✅ **关键信息匹配**
- 开票日期一致
- 金额一致（允许小数点误差）
- 校验码正确（增值税专用发票）

✅ **销售方信息正常**
- 销售方税号有效
- 销售方未被列入异常名单
- 销售方开票资格正常

### 2. 验真失败情况

❌ **发票不存在** (NOT_FOUND)
- 税务系统查无此票
- 可能是假发票或录入错误

❌ **发票已作废** (VOID)
- 开票方已作废该发票
- 不能用于报销

❌ **发票已红冲** (RED)
- 发票已被冲红（退货/退款）
- 不能用于报销

❌ **信息不匹配** (MISMATCH)
- 金额、日期等关键信息与税务系统不一致
- 可能是发票被篡改

❌ **异常发票** (ABNORMAL)
- 被税务机关标记为异常
- 销售方存在税务风险

❌ **重复使用** (DUPLICATE)
- 该发票已在系统中报销过
- 防止重复报销

## 二、推荐的税务服务商

### 1. 百望云（推荐）⭐⭐⭐⭐⭐

**优势：**
- 直连国家税务总局系统
- 无查询次数限制
- 响应速度快（平均200ms）
- 支持批量查询
- 价格合理（约0.1-0.3元/次）

**申请流程：**
1. 访问：https://www.baiwang.com/
2. 注册企业账号
3. 申请"发票查验API"服务
4. 获取 APP_KEY 和 APP_SECRET
5. 充值后即可使用

**配置方法：**
```bash
# 在 .env 文件中添加
BAIWANG_APP_KEY=your_app_key_here
BAIWANG_APP_SECRET=your_app_secret_here
```

### 2. 航天信息 ⭐⭐⭐⭐

**优势：**
- 老牌税务服务商
- 稳定性好
- 支持全国发票查验

**申请流程：**
1. 访问：https://www.aisino.com/
2. 联系销售申请API服务
3. 签订合同并获取API密钥

**配置方法：**
```bash
# 在 .env 文件中添加
HANGTIAN_API_KEY=your_api_key_here
```

**限制：**
- 每天每张发票最多查询5次
- 需要手动输入验证码
- 有IP访问频率限制
- 不适合批量查询

**适用场景：**
- 仅适合手工查验少量发票
- 不推荐用于系统自动化对接

## 四、验真结果处理

### 1. 验真通过 (PASSED)

```json
{
  "result_status": "PASSED",
  "invoice_status": "NORMAL",
  "result_message": "发票验真通过",
  "invoice_data": {
    "invoice_code": "044031900111",
    "invoice_number": "12345678",
    "invoice_date": "2024-01-15",
    "amount": "1000.00",
    "seller_name": "某某公司",
    "seller_tax_no": "91110000XXXXXXXXXX",
    "buyer_name": "购买方公司",
    "buyer_tax_no": "91110000YYYYYYYYYY"
  }
}
```

**处理建议：**
- ✅ 允许继续审批流程
- 记录验真成功日志
- 保存发票详细信息

### 2. 验真失败 (FAILED)

```json
{
  "result_status": "FAILED",
  "invoice_status": "VOID",
  "result_message": "发票已作废",
  "result_code": "VOID"
}
```

**处理建议：**
- ❌ 阻止继续审批
- 标记为高风险
- 通知提交人补充材料
- 记录失败原因

### 3. 系统异常 (ERROR)

```json
{
  "result_status": "FAILED",
  "result_code": "NETWORK_ERROR",
  "result_message": "网络请求失败"
}
```

**处理建议：**
- ⚠️ 允许人工审核
- 记录异常日志
- 稍后重试验真

### 成本优化建议

1. **缓存验真结果**
   - 同一张发票24小时内不重复查询
   - 减少重复验真成本

2. **批量查询**
   - 使用批量接口降低单价
   - 夜间批量处理降低峰值

3. **分级验真**
   - 低金额发票：仅验真一次
   - 高金额发票：多次验真确认

## 六、安全建议

1. **API密钥保护**
   - 不要将密钥提交到代码仓库
   - 使用环境变量或密钥管理服务
   - 定期轮换密钥

2. **请求签名**
   - 使用HMAC或RSA签名
   - 防止请求被篡改

3. **HTTPS加密**
   - 所有API请求使用HTTPS
   - 验证服务器证书

4. **日志脱敏**
   - 记录日志时脱敏敏感信息
   - 不记录完整的税号、金额等

## 八、常见问题

### Q1: 验真失败率高怎么办？

**可能原因：**
- 发票信息录入错误（OCR识别错误）
- 发票确实有问题
- 网络不稳定

**解决方案：**
- 提高OCR识别准确率
- 允许人工修正后重新验真
- 增加重试机制

### Q2: 验真速度慢怎么办？

**优化方案：**
- 使用异步验真
- 批量查询
- 增加超时重试
- 使用CDN加速

### Q3: 成本太高怎么办？

**降本方案：**
- 缓存验真结果
- 分级验真策略
- 批量查询优惠
- 选择性验真（仅验高风险发票）
