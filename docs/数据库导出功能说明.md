# 真实数据库导出功能交付文档

## 功能概述

自动识别项目中的真实数据库（排除 demo/clean 库），并导出为可在 Navicat 中直接打开的 SQLite 文件。

## 使用方法

### 一键导出（推荐）

双击运行导出脚本：

```cmd
scripts\export_real_db.bat
```

脚本会自动完成以下操作：
1. 扫描项目配置，识别真实数据库
2. 排除所有 demo/clean 数据库
3. 复制到 `exports/database_real_for_navicat.db`
4. 显示 Navicat 导入步骤

### 在 Navicat 中打开

导出完成后，按照终端提示的步骤操作：

1. 打开 Navicat Premium 或 Navicat for SQLite
2. 点击「连接」→ 选择「SQLite」
3. 设置连接：
   - 连接名：`DeepAudit_Pro_Real`
   - 数据库文件：`C:\Users\画桦\Desktop\DeepAudit_Pro\exports\database_real_for_navicat.db`
4. 点击「测试连接」→「确定」
5. 双击连接即可查看数据

## 识别规则

脚本按以下优先级识别真实数据库：

### 1. 环境变量（最高优先级）
- 检查 `DATABASE_URL` 或 `SQLALCHEMY_DATABASE_URI`
- 排除包含 `demo` 或 `clean` 的路径

### 2. 配置文件默认值
- 解析 `core/settings.py` 中的默认配置
- 默认为 `sqlite:///database.db`

### 3. 兜底规则
- 扫描项目中所有 `.db` 文件
- 排除文件名包含 `demo` 或 `clean` 的数据库
- 选择最近修改的数据库

## 安全保证

✅ **只读操作**：脚本只复制数据库，不会修改原文件  
✅ **自动备份**：如果导出文件已存在，会自动备份为 `database_real_for_navicat_YYYYMMDD_HHMMSS.db`  
✅ **严格过滤**：绝对不会导出 demo/clean 数据库  
✅ **验证机制**：导出后自动验证数据库完整性

## 常见问题

### Q: 如何确认导出的是真实数据库？
A: 脚本会在终端明确显示识别依据和数据库路径，请仔细核对。

### Q: 导出文件可以删除吗？
A: 可以。导出文件只是副本，删除不会影响原数据库。

### Q: 支持其他数据库工具吗？
A: 支持。导出的是标准 SQLite 文件，可用任何 SQLite 工具打开（DB Browser、DBeaver 等）。

### Q: 如何导出 demo 数据库？
A: 本脚本专门用于导出真实库。如需查看 demo 库，请直接在 Navicat 中打开 `database_demo.db`。

## 技术实现

- **语言**: Python 3.8+
- **依赖**: 仅使用标准库（os, shutil, sqlite3, datetime, pathlib）
- **兼容性**: Windows 10/11
- **编码**: UTF-8（自动处理 Windows 控制台编码问题）
