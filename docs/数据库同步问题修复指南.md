# 数据库同步问题修复指南

## 问题根因

项目中存在**3个数据库文件**，导致写入和读取不同步：

1. **`database.db`** (根目录) - 默认配置使用的数据库
2. **`instance/database.db`** - Flask 实例目录（可能未使用）
3. **`exports/database_real_for_navicat.db`** - 静态导出副本（不会自动同步）

**核心问题**：
- Flask 应用写入的是 `database.db`（根目录）
- Navicat 打开的是 `exports/database_real_for_navicat.db`（静态副本）
- 两者不是同一个文件，所以看不到实时更新

## 数据库配置优先级

系统按以下优先级确定数据库路径：

1. **环境变量 `DATABASE_URL`**（最高优先级）
   - 格式：`sqlite:///database.db`（相对路径）
   - 格式：`sqlite:///C:/absolute/path/database.db`（绝对路径）

2. **环境变量 `SQLALCHEMY_DATABASE_URI`**（次优先级）
   - 同上格式

3. **`core/settings.py` 默认值**（兜底）
   - 默认：`sqlite:///database.db`（解析为根目录）

4. **`utils/db.py` 的 `DB_PATH`**
   - 从上述配置解析而来

## 修复方案（二选一）

### 方案 A：Navicat 直接打开真实数据库（推荐）

**优点**：实时同步，无需手动导出  
**缺点**：需要关闭 Navicat 才能运行 Flask 应用（SQLite 文件锁）

**操作步骤**：

1. **运行诊断脚本，找到真实数据库路径**：
   ```bash
   scripts\diagnose_db_path.py
   ```
   或
   ```bash
   scripts\fix_db_sync.bat
   ```

2. **在 Navicat 中打开真实数据库**：
   - 关闭当前连接（如果打开了 exports/ 下的副本）
   - 新建连接 → SQLite
   - 连接名：`DeepAudit_Pro_Real`
   - 数据库文件：选择诊断脚本输出的**真实数据库路径**
     - 通常是：`C:\Users\画桦\Desktop\DeepAudit_Pro\database.db`
   - 测试连接 → 确定

3. **使用注意事项**：
   - **查看数据时**：关闭 Flask 应用，打开 Navicat
   - **运行应用时**：关闭 Navicat，启动 Flask
   - SQLite 不支持多进程同时写入

### 方案 B：使用导出副本（需手动同步）

**优点**：可以同时运行 Flask 和 Navicat  
**缺点**：每次修改后需手动导出

**操作步骤**：

1. **每次需要查看最新数据时，运行导出脚本**：
   ```bash
   scripts\export_real_db.bat
   ```
   或
   ```bash
   python scripts\export_real_db.py
   ```

2. **在 Navicat 中打开导出的副本**：
   - 数据库文件：`C:\Users\画桦\Desktop\DeepAudit_Pro\exports\database_real_for_navicat.db`

3. **刷新 Navicat 数据**：
   - 右键连接 → 刷新
   - 或关闭重新打开连接

## 常见问题

### Q1: 为什么 Navicat 看不到最新数据？

**A**: 检查以下几点：
1. Navicat 打开的文件路径是否正确（运行 `diagnose_db_path.py` 确认）
2. 如果使用导出副本，是否重新运行了导出脚本
3. Navicat 是否刷新了连接

### Q2: 如何确认 Flask 写入的是哪个数据库？

**A**: 运行 `python scripts\verify_db_write.py`，会显示写入目标并执行测试写入。

### Q3: 可以同时运行 Flask 和 Navicat 吗？

**A**: 
- **方案 A（直接打开真实库）**：不可以，SQLite 文件锁冲突
- **方案 B（使用导出副本）**：可以，但需要手动同步

### Q4: 如何删除多余的数据库文件？

**A**: 
1. 先运行 `diagnose_db_path.py` 确认哪个是真实库
2. 备份真实库
3. 删除其他副本（如 `instance/database.db`）
4. 保留 `exports/` 下的副本作为导出目标

## 技术细节

### SQLite 文件锁机制

SQLite 使用文件锁来保证数据一致性：
- **读锁**：多个进程可以同时读
- **写锁**：只有一个进程可以写，且会阻塞其他读写

因此：
- Flask 运行时（有写操作），Navicat 可能无法打开
- Navicat 打开时（持有读锁），Flask 写入可能被阻塞

### 相对路径解析

`sqlite:///database.db` 的解析规则：
- 相对于**项目根目录**（`PROJECT_ROOT`）
- 不是相对于当前工作目录（`cwd`）
- 由 `utils/db.py` 的 `_get_db_path_from_env()` 处理

### 配置优先级实现

```python
# utils/db.py
def _get_db_path_from_env() -> str:
    # 1. DATABASE_URL（最高优先级）
    database_url = os.getenv("DATABASE_URL", "").strip()
    if database_url and database_url.startswith("sqlite:///"):
        return parse_and_resolve(database_url)
    
    # 2. DB_PATH（兼容旧版）
    db_path = os.getenv("DB_PATH", "").strip()
    if db_path:
        return os.path.abspath(db_path)
    
    # 3. 默认值
    return os.path.abspath(str(PROJECT_ROOT / "database.db"))
```
