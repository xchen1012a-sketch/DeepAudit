双击运行：
```

```

**这就是 Flask 实际写入的数据库文件！**

### 第三步：在 Navicat 中打开这个文件

1. 打开 Navicat
2. 新建连接 → SQLite
3. 连接名：`DeepAudit_Pro_Real`

5. 测试连接 → 确定

### 第四步：验证同步

1. 关闭 Navicat
2. 启动 Flask 应用：`python app.py`

5. 打开 Navicat，查看数据是否更新

✅ **如果能看到新数据，问题解决！**

## 📋 问题根因（已确认）

| 项目 | 路径 | 说明 |
|------|------|------|
| Flask 写入 | `database.db`（根目录） | 真实数据库 |
| Navicat 打开 | `exports/database_real_for_navicat.db` | 静态副本 |
| 结果 | ❌ 不同步 | 两个不同的文件 |

---

## 🎯 两种解决方案

### 方案 A：直接打开真实库（推荐）

**优点**：✅ 实时同步  
**缺点**：⚠️ 查看数据时需关闭 Flask

**Navicat 打开**：`C:\Users\画桦\Desktop\DeepAudit_Pro\database.db`

**使用规则**：
- 查看数据 → 关闭 Flask → 打开 Navicat
- 运行应用 → 关闭 Navicat → 启动 Flask

### 方案 B：使用导出副本

**优点**：✅ 可以同时运行  
**缺点**：⚠️ 需手动导出

**Navicat 打开**：`C:\Users\画桦\Desktop\DeepAudit_Pro\exports\database_real_for_navicat.db`

**每次查看最新数据前运行**：
```bash
python scripts\export_real_db.py
```

然后在 Navicat 中刷新连接。

## ❓ 常见问题

### Q: 为什么 Navicat 看不到最新数据？

**A**: 检查 Navicat 打开的文件路径是否正确。运行诊断工具确认：
```bash
python scripts\diagnose_db_path.py
```

### Q: 可以同时运行 Flask 和 Navicat 吗？

**A**: 
- 方案 A（直接打开真实库）：❌ 不可以（SQLite 文件锁）
- 方案 B（使用导出副本）：✅ 可以（但需手动同步）

### Q: 如何切换数据库位置？

**A**: 在项目根目录创建 `.env` 文件：
```env
DATABASE_URL=sqlite:///D:/MyData/deepaudit.db
```
